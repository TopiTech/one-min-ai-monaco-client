# one-min-ai-monaco-client コードレビュー計画

## 概要
BFF（Backend-for-Frontend）パターンのMonacoエディタベースの1min.aiクライアント。バックエンド（Express）＋フロントエンド（SPA）構成。1min.ai最新ドキュメントと照合し、バグ修正・セキュリティ・UI/UX・機能拡張の観点から包括的レビューを実施する。

---

## Phase 1: クリティカルバグ修正（優先度: 最高）

### 1.1 APIクライアント - lastError未定義バグ
- **ファイル**: `utils/api-client.js:68-125`
- **問題**: 全リトライが429の場合、`lastError`が`undefined`のまま`lastError.message`を呼んでTypeError発生
- **修正**: `let lastError = new Error('All retry attempts failed');` で初期化

### 1.2 APIクライアント - Retry-After NaNバグ
- **ファイル**: `utils/api-client.js:82-83`
- **問題**: Retry-Afterヘッダーが日付文字列の場合`parseInt`がNaN→即時リトライの無限ループ
- **修正**: `parseInt`結果を`isNaN`チェックし、フォールバック値を使用

### 1.3 チャットAPI - historySettingsフィールド不整合
- **ファイル**: `routes/ai.js:26-28`
- **問題**: `historySettings: { history: Boolean(history) }` だが1min.ai APIは`{ isMixed: boolean, historyMessageLimit: number }`を要求。`history`は無効フィールド
- **修正**: `{ isMixed: false, historyMessageLimit: 10 }` に変更

### 1.4 エラーハンドラ - アップストリーム詳細漏洩
- **ファイル**: `server.js:110-119`（推定行）
- **問題**: `details: err.payload`で1min.ai内部エラー詳細をクライアントに返却
- **修正**: `NODE_ENV === 'development'` の場合のみ`details`を含める

### 1.5 セッションID予測可能
- **ファイル**: `routes/agent.js:19`（推定）
- **問題**: `session-${Date.now()}`でID生成→容易に予測・列挙可能
- **修正**: `crypto.randomUUID()` をフォールバックに使用

### 1.6 承認トークンの暗号学的ランダム性不足
- **ファイル**: `routes/agent.js:99`（推定）
- **問題**: `Math.random()`使用→暗号学的に安全でない
- **修正**: `crypto.randomUUID()` に変更

## Phase 2: セキュリティ修正（優先度: 高）

### 2.1 XSS脆弱性 - フロントエンド
- **ファイル**: `public/app.js:279, 898-901`
- **問題**: サーバー提供データを`innerHTML`に直接挿入
- **修正**: `textContent`使用、またはDOM APIで要素を構築

### 2.2 XSS脆弱性 - モデルピッカー
- **ファイル**: `public/js/models.js:72`
- **問題**: `innerHTML`にモデル名・IDを挿入
- **修正**: `textContent`に変更

### 2.3 コマンド実行 - ブロックリストの脆弱性
- **ファイル**: `services/command-runner.js:12-36, 92`
- **問題**: ブロックリストのみで危険コマンドをブロック。`curl`, `wget`, `nc`, `python -c`, `node -e`, `docker`等が未カバー
- **修正**: 
  - 短期: 追加パターンのブロック（`curl`, `wget`, `nc`, `python`, `node -e`, `docker`, `kubectl`, `git push --force`, `rm`, Windows固有コマンド）
  - 長期: アローリストモードへの移行オプション追加

### 2.4 CSP `unsafe-inline`
- **ファイル**: `server.js:29-30`（推定）
- **問題**: `scriptSrc`に`'unsafe-inline'`が含まれXSSリスク増大
- **修正**: Monaco Web Workerがblob URLで動作するか確認し、不要なら削除

### 2.5 CDNスクリプトのSRI欠如
- **ファイル**: `public/index.html:473-474`（推定）
- **問題**: `marked`と`monaco-editor`にSubresource Integrityハッシュなし
- **修正**: `integrity`属性を追加

### 2.6 markedライブラリのバージョン固定なし
- **ファイル**: `public/index.html:473`（推定）
- **問題**: `marked/marked.min.js`がバージョン固定されていない
- **修正**: 確認済みバージョンで固定（例: `marked@15.0.0`）

### 2.7 Multer MIMEタイプフィルタなし
- **ファイル**: `server.js:20-23`（推定）
- **問題**: アップロードファイルのMIMEタイプ検証なし
- **修正**: `fileFilter`で許可するMIMEタイプを指定

### 2.8 入力サイズ制限なし
- **ファイル**: `routes/ai.js:140-203`
- **問題**: `code`, `instruction`, `prompt`フィールドにサイズ制限なし。メガバイト単位の入力が可能
- **修正**: 各フィールドに最大文字数制限を追加（例: コード10万文字、プロンプト5万文字）

## Phase 3: バックエンド品質改善（優先度: 中）

### 3.1 ファイル読み込み前サイズチェック
- **ファイル**: `routes/fs.js:146-162`（推定）
- **問題**: `readFile`前に`stat`でファイルサイズを確認しないため、巨大ファイルでOOMリスク
- **修正**: `stat.size`を確認し、閾値（例: 10MB）超ならエラー返却

### 3.2 ディレクトリリストのページネーション
- **ファイル**: `routes/fs.js:113-141`（推定）
- **問題**: `readdir`で全エントリを返す。巨大ディレクトリでペイロード过大
- **修正**: `offset`/`limit`パラメータ追加、または上限件数設定

### 3.3 ロガーの同期I/O
- **ファイル**: `utils/logger.js:62`（推定）
- **問題**: `appendFileSync`でイベントループをブロック
- **修正**: `fs.promises.appendFile`またはWriteStreamに変更

### 3.4 PROJECT_ROOTのCWD依存
- **ファイル**: `utils/fs-guard.js:4`（推定）
- **問題**: `path.resolve('.')`でCWD依存。異なるディレクトリから起動すると問題
- **修正**: `import.meta.url`でファイル自身のディレクトリを取得（`server.js`の実装を踏襲）

### 3.5 シンボリックリンク対策の不備
- **ファイル**: `utils/fs-guard.js:54-59`（推定）
- **問題**: 存在しないパスはシンボリックリンク解決をスキップ。書き込み時にサンドボックス脱出可能
- **修正**: 存在しないパスの親ディレクトリを遡って解決

### 3.6 コードフェンス除去の重複
- **ファイル**: `routes/ai.js:190-197, 255-262`
- **問題**: markdown コードフェンス除去ロジックが`autocomplete`と`inline-chat`で完全コピー
- **修正**: 共通ユーティリティ関数に抽出

### 3.7 モデルバリデーションなし
- **ファイル**: `routes/ai.js:19, 128, 178, 243`
- **問題**: クライアントから任意のモデル名が転送される
- **修正**: `config/models.js`のモデルリストで検証

## Phase 4: フロントエンド品質改善（優先度: 中）

### 4.1 メモリリーク - Object URL
- **ファイル**: `public/app.js:146, 240`
- **問題**: `URL.createObjectURL()`を`URL.revokeObjectURL()`で解放しない
- **修正**: 添付ファイルクリア時に`revokeObjectURL()`呼び出し

### 4.2 メモリリーク - Monacoモデル
- **ファイル**: `public/app.js:658-661`
- **問題**: ファイルごとにMonacoモデルを作成するが破棄しない
- **修正**: ファイル切替時に`model.dispose()`呼び出し

### 4.3 メモリリーク - DOM無限成長
- **ファイル**: `public/app.js:91-92, 280`
- **問題**: チャットログ・画像ギャラリーが無制限にDOM追加
- **修正**: 仮想スクロール、または上限件数（例: 200件）で古い要素を削除

### 4.4 送信ボタンの重複送信
- **ファイル**: `public/app.js:206`
- **問題**: リクエスト中も送信ボタンがクリック可能
- **修正**: リクエスト中はボタンを`disabled`にし、`finally`で復元

### 4.5 画像アップロードの並列化
- **ファイル**: `public/app.js:165`
- **問題**: 添付ファイルを逐次アップロード
- **修正**: `Promise.all()`で並列アップロード

### 4.6 フロントエンドAPIタイムアウトなし
- **ファイル**: `public/js/api.js:10`
- **問題**: `fetch()`にタイムアウトなし。サーバーハング時にUIフリーズ
- **修正**: `AbortController` + `setTimeout`でタイムアウト追加

### 4.7 ステータスバーの状態競合
- **ファイル**: `public/js/api.js:1, 3-8`
- **問題**: 複数同時リクエストでステータスが相互上書き
- **修正**: キューイングまたはリクエスト固有のステータス表示

### 4.8 モデル読み込み失敗時のフォールバック
- **ファイル**: `public/js/models.js:14`（推定）
- **問題**: `loadModels()`失敗時にユーザーに通知なし
- **修正**: トースト通知でエラー表示

### 4.9 ドロップダウンリスナー蓄積
- **ファイル**: `public/js/models.js:132-139`（推定）
- **問題**: ドロップダウン内クリックごとに新しい`{ once: true }`リスナー追加
- **修正**: 単一の永続リスナーに変更

### 4.10 saveFileステータス競合
- **ファイル**: `public/app.js:695-698`
- **問題**: 2秒後にステータスをリセットするが、連続保存で競合
- **修正**: タイマーキャンセル機能追加

## Phase 6: 機能拡張（1min.ai最新API活用）

### 6.1 ストリーミングチャット対応
- **関連ファイル**: `routes/ai.js`, `public/app.js`, `public/js/api.js`
- **内容**: 1min.ai APIは`?isStreaming=true`でSSEストリーミングをサポート。現在は非ストリーミングのみ
- **実装**:
  - バックエンド: `callOneMin`に`raw: true`オプションでSSEレスポンスをストリーミーム
  - フロントエンド: `EventSource`または`fetch`の`ReadableStream`でチャンク受信
  - UI: チャットメッセージをリアルタイムで更新

### 6.2 AIメモリー機能
- **関連ファイル**: `routes/ai.js:29`
- **内容**: `withMemories: false` hardcode。APIは`withMemories: true`で会話間メモリーをサポート
- **実装**: UIにトグルを追加し、`withMemories`を設定可能に

### 6.3 ブランドボイス機能
- **関連ファイル**: `routes/ai.js`, `public/app.js`
- **内容**: APIは`brandVoiceId`でカスタムトーン/スタイルをサポート
- **実装**: 設定画面にブランドボイス選択UIを追加

### 6.4 YouTube動画チャット
- **関連ファイル**: `routes/ai.js`, `public/app.js`
- **内容**: プロンプトにYouTube URLを含めると自動的にトランスクリプトを取得して文脈として使用
- **実装**: チャット入力でYouTube URL検出→フロントエンドでフィードバック表示

### 6.5 画像バリアター
- **関連ファイル**: 新規ルート追加
- **内容**: `IMAGE_VARIATOR`タイプで既存画像のバリエーション生成（APIでサポート）
- **実装**: 画像ビューに「バリエーション生成」タブを追加

### 6.6 ファイル添付チャット
- **関連ファイル**: `routes/ai.js`, `public/app.js`
- **内容**: チャットにPDF等のファイルを添付可能（Asset API経由）
- **実装**: チャットにファイルアップロードボタン追加→Asset APIでアップロード→`attachments.files`に設定

### 6.7 インラインチャットの改善
- **関連ファイル**: `public/app.js:536-544`
- **内容**: 現在はカーソル位置にコードを挿入。AIが全文変更を返した場合の処理が不完全
- **実装**: diffベースの適用、または置換範囲の明示的指定

## Phase 7: テスト・検証

### 7.1 既存テストの実行と修正

```bash
npm test
```

### 7.2 新規テスト追加
- `api-client.test.js`: `lastError`初期化、Retry-After NaN処理のテスト追加

1. サーバー起動: `npm run dev`
2. ブラウザで `http://localhost:3000` を開く
3. チャット機能: メッセージ送信→レスポンス確認→会話履歴確認

5. コーディング: ファイル作成→AI補助→保存→再読み込み

7. エージェント: コマンド実行→承認フロー確認
8. レスポンシブ: モバイル表示でレイアウト確認

### 7.4 セキュリティテスト
- パストラバーサル: `../../etc/passwd`のようなパスでアクセス試行
- XSS: ファイル名にHTML/JSを含めて動作確認
- コマンドインジェクション: 危険コマンド入力でブロック確認
- レート制限: 連続リクエストで制限動作確認
