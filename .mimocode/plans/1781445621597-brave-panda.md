# Code Review: 1min.ai Monaco Client

## Executive Summary

1min.ai Monaco Client は、1min.ai API を活用したブラウザベースの AI クライアントです。Monaco Editor によるコーディング支援、チャット、画像生成機能を提供しています。以下のレビューでは、最新の 1min.ai API ドキュメント (2026年6月時点) と照らし合わせて、不具合修正、機能拡張、UI 改善、バックエンド改善の観点から問題点を洗い出しました。

---

## P0: Critical Bugs (即時修正推奨)

### 1. 画像生成 API ペイロードの仕様不一致
**ファイル**: `routes/ai.js:75-104`

**問題**: `/images/generate` エンドポイントが `num_outputs` と `aspect_ratio` を使用しているが、デフォルトモデル `gpt-image-2` の仕様では以下のパラメータが正しい:
- `size` (例: "1024x1024") — `aspect_ratio` ではなくサイズ文字列
- `n` — `num_outputs` ではなく `n`
- `quality` — 必須パラメータだが未送信

**影響**: デフォルトモデル `gpt-image-2` で画像生成すると、パラメータが無視され予期しない結果になる。

**修正案**:
```javascript
// 現在のコード (routes/ai.js:89-94)
promptObject: {
  prompt: String(prompt),
  num_outputs: Number(num_outputs) || 1,  // ❌ gpt-image-2 では `n`
  aspect_ratio,                            // ❌ gpt-image-2 では `size`
  output_format,
}

// 修正後
promptObject: {
  prompt: String(prompt),
  n: Number(num_outputs) || 1,
  size: aspectRatioToSize(aspect_ratio),   // "1:1" → "1024x1024"
  quality: "medium",
  output_format,
}
```

### 2. 設定画面のデフォルト画像モデル表示が古い
**ファイル**: `public/index.html:936`

**問題**: 設定画面に `DEFAULT_IMAGE_MODEL: black-forest-labs/flux-schnell` と表示されているが、サーバーのデフォルトは `gpt-image-2`。

**影響**: ユーザーが混乱する。

**修正**: 表示を `gpt-image-2` に更新。

## P1: High Priority Issues

### 3. ストリーミング未対応
**ファイル**: `routes/ai.js:47-51`, `public/app.js:227-238`

**問題**: 1min.ai API は `?isStreaming=true` による SSE ストリーミングをサポートしているが、このプロジェクトでは非ストリーミングのみ使用。

**影響**: チャット応答の待ち時間が長く、ユーザー体験が劣化。

**推奨対応**:
- バックエンド: `/api/chat/stream` エンドポイントを追加し、SSE プロキシを実装
- フロントエンド: `EventSource` または `fetch` + `ReadableStream` で受信し、テキストを逐次表示

### 4. ファイル添付未対応
**ファイル**: `public/app.js:140-163`, `public/index.html:125-131`

**問題**: チャットは画像添付のみ対応。1min.ai API は `attachments.files` で PDF/ドキュメント添付をサポート。

**影響**: PDF 分析やドキュメントQ&Aができない。

**修正案**:
- ファイル入力の `accept` を `image/*` から `*` に拡張
- `uploadAttachments()` を汎用化し、ファイルタイプに応じて `type: "image"` または `type: "file"` を設定
- バックエンドの `/api/chat` ルートで `attachments.files` を渡す

### 5. コード生成 API の `conversationId` 設定
**ファイル**: `routes/ai.js:184`

**問題**: 1min.ai の CODE_GENERATOR ドキュメントでは `conversationId` が必須。現在は `"CODE_GENERATOR"` をハードコード。

**影響**: 機能自体は動作するが、会話コンテキストが保持されない。

**改善**: ユーザーが conversationId を指定できるようにする、または自動生成。

### 6. エージェントループが最後のメッセージのみ送信
**ファイル**: `public/app.js:1118-1119`

**問題**:
```javascript
body: JSON.stringify({
  prompt: agentConversationHistory[agentConversationHistory.length - 1].content,
  // ...
})
```
`history: true` を設定しているが、サーバーに送信するのは最後のメッセージのみ。`agentConversationHistory` 配列はフロントエンドで管理されているが、サーバーには渡されていない。

**影響**: エージェントが会話コンテキストを失い、同じことを繰り返す可能性がある。

**修正案**: `agentConversationHistory` 全体をサーバーに渡すか、サーバー側で会話を管理。

### 7. モデル設定の不足
**ファイル**: `config/models.js`

**問題**: 最新の 1min.ai API で利用可能なモデルが欠落:
- **画像生成**: Magic Art 7.0, Flux 2 Pro/Dev/Flex/Max/Klein 4B/9B, Gemini 3 Pro Image Preview, Gemini 3.1 Flash Image, Grok-2 Image, Qwen Image, Recraft
- **画像編集**: Flux 2 Pro/Dev/Flex/Max/Klein 4B, Gemini 3 Pro/Flash, Qwen Image Edit Plus
- **音声**: OpenAI TTS, Google TTS, ElevenLabs TTS, Qwen3 TTS Flash
- **動画**: Luma AI, Kling AI, Veo3, Sora, Hailuo AI, Pika AI

## P2: Medium Priority Improvements

### 8. 音声・動画・ライティング機能の未実装
**ファイル**: 未存在（新規追加が必要）

**問題**: 1min.ai API は以下の機能を提供しているが、このプロジェクトでは未実装:
- Text-to-Speech (OpenAI, Google, ElevenLabs)
- Speech-to-Text
- Text-to-Video (Luma, Kling, Veo3, Sora, etc.)
- Content Generator (ライティング)

**推奨**: 新しいビューやサイドバーセクションとして追加。

### 9. 画像加工機能の不足
**ファイル**: `routes/ai.js`, `public/app.js`

**問題**: 画像テキスト編集のみ対応。以下が未実装:
- Image Variator (画像バリアーション)
- Image Upscaler (画像アップスケール)
- Background Remover (背景除去)
- Image to Prompt (画像→プロンプト変換)

### 10. 会話履歴の永続化なし
**ファイル**: `public/app.js:789`

**問題**: エージェントの会話履歴はインメモリのみ。ページリロードで消失。

**推奨対応**:
- LocalStorage にセッション情報を保存
- サーバー側にセッション永続化レイヤーを追加

### 11. エージェントのトークン制限管理なし
**ファイル**: `public/app.js:789`

**問題**: `agentConversationHistory` が無制限に增长し、API のトークン制限を超える可能性。

**推奨対応**:
- トークン数を推定し、古いメッセージを要約或は削除
- 429 エラー時の自動リトライとバックオフ

### 12. `brandVoiceId` サポート未実装
**ファイル**: `routes/ai.js:13-55`

**問題**: 1min.ai Chat API は `brandVoiceId` パラメータをサポートしているが、未実装。

**推奨**: チャット設定にブランドボイス選択 UI を追加。

### 13. `historySettings.isMixed` がハードコード
**ファイル**: `routes/ai.js:37`

**問題**: `isMixed: false` にハードコード。ユーザーが異なるモデルのコンテキストを混ぜる設定をできない。

**修正**: フロントエンドにチェックボックスを追加し、設定可能にする。

### 14. 画像サイズのバリデーション不足
**ファイル**: `routes/ai.js:112`

**問題**: GPT Image 2 の `size` パラメータは `WxH` 形式で、幅と高さが16の倍数、合計ピクセル数に制限がある。バリデーションなし。

**推奨**: サーバー側でサイズ文字列のバリデーションを追加。

## P3: Low Priority / UX Improvements

### 15. チャット中のローディング表示なし
**ファイル**: `public/app.js:211-256`

**問題**: チャット送信後に AI の応答を待っている間、ユーザーにフィードバックがない。

**推奨**: タイピングアニメーションや「考え中...」表示を追加。

### 16. インライン補間のデバウンスが長い
**ファイル**: `public/app.js:457`

**問題**: 750ms のデバウンス。よりresponsive な体験が可能。

**推奨**: 300-500ms に短縮、またはユーザー設定可能にする。

### 17. インラインスタイルの多用
**ファイル**: `public/index.html`

**問題**: 多くの要素にインラインスタイル。保守性低下。

**推奨**: CSS クラスに移行。

### 18. CORS 設定なし
**ファイル**: `server.js`

**問題**: CORS ヘッダーが未設定。異なるオリジンからのアクセスが必要な場合に問題に。

**推奨**: 開発用に `cors` ミドルウェアを追加。

### 19. レート制限情報のフロントエンド表示なし
**ファイル**: `server.js:73-87`

**問題**: レート制限設定がサーバー側のみ。ユーザーが制限に達した場合のフィードバックがない。

**推奨**: レート制限エラー時にユーザーに通知。

### 20. チャットの会話リスト機能なし
**ファイル**: `public/app.js`

**問題**: 作成した会話を一覧表示する機能がない。手動で UUID を入力する必要がある。

**推奨**: サイドバーに会話リストを表示し、選択して切り替えられるようにする。
